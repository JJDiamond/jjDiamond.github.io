单张图片

单张图片还是比较容易实现，在这里的处理方式就是使用背景图片

单张或者多张图片任意宽高垂直居中 display:table-cell

多行文字垂直居中，使用span标签

该方法类似于第二种图片垂直居中。

多图片实现垂直居中的第三种方法，li标签

该方法使用line-height样式，使用font-size控制居中。

绝对定位方式垂直居中

该方法的局限在于需要知晓中间垂直元素的高度和宽度，设置margin-left和margin-top的时候才能计算出来具体的数值。

关于meta标签常用问题

meta标签常用属性与属性值的不同作用

在开发过程中，遇到过很多问题，通过添加meta标签属性就可以简单的解决问题，刚好抽空对于meta标签做出总结，如下：

meta

http-equiv 属性的 Content-Type 值（显示字符集的设定）

charset属性大家应该都比较熟悉了，HTML 页面所使用的字符集为 utf-8，在这里就不做过多的解释了。

name 属性的 viewport 值（移动屏幕的缩放）

顾名思义：
1、强制让文档与设备的宽度保持 1:1 ；
2、文档最大的宽度和最小宽度比例都是1.0（ initial-scale 初始刻度值和 maximum-scale 最大刻度值）；
3、user-scalable 定义用户是否可以手动缩放（ no 为不缩放），使页面固定设备上面的大小；

  • width – // viewport 的宽度 （范围从 200 到 10,000，默认为 980 像素）
  • height – // viewport 的高度 （范围从 223 到 10,000 ）
  • initial-scale – // 初始的缩放比例 （范围从 > 0 到 10）
  • minimum-scale – // 允许用户缩放到的最小比例
  • maximum-scale – // 允许用户缩放到的最大比例

  • user-scalable – // 用户是否可以手动缩放 (no，yes)

    name 属性的 format-detection 值（忽略页面中的数字识别为电话号码）

  • 使设备浏览网页时对数字不启用电话功能（不同设备解释不同，iTouch 点击数字为存入联系人，iPhone 为拨打电话），忽略将页面中的数字识别为电话号码。

  • 若需要启用电话功能将 telephone=yes 即可，若在页面上面有 Google Maps, iTunes 和 YouTube 的链接会在ios设备上打开相应的程序组件。

    name 属性的 apple-mobile-web-app-capable 值（网站开启对 web app 程序的支持）

  • 网站开启对 web app 程序的支持。

  • 该 meta 可以看出内容为“苹果设备 web 应用程序 xx”，就是说该 meta 是专门定义 web 应用的。

    name 属性的 apple-mobile-web-app-status-bar-style 值（改变顶部状态条的颜色）

  • 在 web app 应用下状态条（屏幕顶部条）的颜色；

  • 默认值为 default（白色），可以定为 black（黑色）和 black-translucent（灰色半透明）；
    注意：若值为“black-translucent”将会占据页面位置，浮在页面上方（会覆盖页面 20px 高度 iphone4 和 itouch4 的 Retina 屏幕为 40px ）。

    name 属性设置作者姓名及联系方式

暂时用过的meta标签属性总结如上，欢迎小伙伴们增加。

js计算误差

js执行算数运算计算误差问题

在一次项目开发中，对于数据实现乘法运算，算出的结果小数点后带了16位小数，用计算器自己算一下，发现数据存在误差，项目要求不能四舍五入，不能向上或者向下取整，于是在网上查找了，寻求解决方法。

误差解释

因为计算机只认识二进制，所以某些数字二进制是无限循环的，例如：0.1=> 0.0001 1001 1001 …无限循环，本身就是无限循环的数据，进行一系列处理后依旧存在循环问题，所以产生了精度问题，c这类语言已经封装好方法来避免，然而js并没有，为此带来不少的麻烦，特别是需要频繁计算的项目，出现bug还不容易发现。
